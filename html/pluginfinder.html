Plugin Finder

for Mod Organizer 2

Introduction

Plugin Finder is a plugin for Mod Organizer 2, allowing users to browse and install other Mod Organizer plugins.

Some of the things you can do with Plugin Finder:

  • Browse a list of plugins and search for them by name.
  • Install plugins directly to Mod Organizer within Plugin Finder.
  • Check for updates to installed plugins and install them within Plugin Finder.
  • Remove plugins form Mod Organizer from within Plugin Finder.

Please note that the plugins found within Plugin Finder are produced and maintained by independent mod authors and may vary in quality. I am not responsible if a plugin installed through Plugin Finder causes damage to your installation or simply does not work. Any issues with plugins found in Plugin Finder should be directed to the independent plugin author.

Installation

To install Plugin Finder, you will first need to download it, either from the Nexus or from the Github releases page for Plugin Finder. Make sure to download the latest version.

Nexus Github

Once you've downloaded the Plugin Finder zip file, you will need to extract the pluginfinder folder from it and place it in the Mod Organizer plugins folder. If Mod Organizer is installed to C:\Mod Organizer\ then the plugins folder is located at C:\Mod Organizer\plugins\

If you have done this correctly, inside C:\Mod Organizer\plugins\pluginfinder\ you should find two folders, shared and pluginfinder as well as a file called __init__.py

Plugin Finder is now installed and will start the next time you run Mod Organizer.

Usage

Plugin Finder can be opened through a new link added to Mod Organizer's tools menu.

This opens the Plugin Finder browser, which allows you to explore the currently available plugins for Mod Organizer.

The search function will only search plugin names for performance reasons.

Adding Plugins

Plugin Finder hosts a central repository containing the plugin's name and the url to a json file containing information on that plugin. To have your plugin added to Plugin Finder, you will need to create a special json file for it.

This json file will contain information about the plugin itself, as well as some basic information on each release/update to your plugin. This file will need to be kept updated each time a new update is released for your plugin. You do not need to keep information on every release, but it is suggested to at least leave in the most working release for each different version of Mod Organizer that you can.

To create this file, there is a generator that can help you to minimise some minor possible mistakes.

Plugin Finder JSON Generator

Alternatively, you can create your json file manually. Below is an example of a possible json file and an explanation of each possible field. 

{
    "Name": "Plugin Template",
    "Author": "Kezyma",
    "Description": "This is a template for a plugin json file to be used with Plugin Finder.",
    "DocsUrl": "https://github.com/Kezyma/ModOrganizer-Plugins/blob/main/readme/pluginfinder/readme.md",
    "NexusUrl": "https://www.nexusmods.com/skyrimspecialedition/mods/59869",
    "GithubUrl": "https://github.com/Kezyma/ModOrganizer-Plugins",
    "Versions": [
        {
        "Version": "1.1.0a",
        "Released": "2021-12-13",
        "MinSupport": "2.4.2",
        "MaxSupport": "2.4.2",
        "MinWorking": "",
        "MaxWorking": "2.4.2",
        "ReleaseNotes": [ "Initial alpha. Demo version." ],
        "DownloadUrl": "https://github.com/Kezyma/ModOrganizer-Plugins/releases/download/pluginfinder/pluginfinder.1.1.0.zip",
        "PluginPath": [ "pluginfinder" ],
        "LocalePath": [],
        "DataPath": [ "data/pluginfinder" ]
        }
    ]
}
Plugin Fields
Field Required Description
Name Yes The name of the plugin to be displayed. Should not be changed once the plugin has been added to Plugin Finder.
Author Yes The name of the plugin author to be displayed.
Description No A short description of this plugin to be displayed, suggested maximum character limit of 350.
DocsUrl No A url to the documentation for your plugin.
NexusUrl No A url to the Nexus page for your plugin.
GithubUrl No A url to the Github page for your plugin.
Versions No An array of versions of this plugin that can be installed through Plugin Finder.
Version Fields
Field Required Description
Version Yes The version number of this version. Should ideally be in the same format as displayed in the settings page of Mod Organizer.
Released Yes The date of this release. Must be in the format yyyy-MM-dd
MinSupport No The minimum version of Mod Organizer that this release has been tested to work with. This release may work on earlier versions, but this has not been tested.
MaxSupport No The maximum version of Mod Organizer that this release has been tested to work with. This release may work on later versions, but this has not been tested.
MinWorking No The minimum version of Mod Organizer that this release works with. This release will not work on earlier versions.
MaxWorking No The maximum version of Mod Organizer that this release works with. This release will not work on later versions.
ReleaseNotes No An array of release notes to be displayed for this version.
DownloadUrl Yes A direct download to the zip file for this release. The url must be to the zip file itself and not to a download page.
PluginPath Yes An array of paths, relative to the root of the download zip. Any files or folders specified will be copied to Mod Organizer's plugins folder.
LocalePath No An array of paths, relative to the root of the download zip. Any files or folders specified will be copied to Mod Organizer's translations folder.
DataPath No An array of paths, relative to Mod Organizer's plugins folder. Any files or folders specified will be deleted when this plugin is uninstalled.

Once you have created your json file, you need to host it on Github with your plugin and get it added to Plugin Finder. There are a few ways to do this.

  • You can open a pull request, editing plugin_directory.json to add your plugin. This will be checked and approved as soon as it is noticed.
  • You can open an issue with the name of your plugin and the link to your hosted json file. It will be added when possible.
  • You can contact me directly on Discord and I will add your plugin when I can. Username: Kezyma#7969

Uninstallation

To remove Plugin Finder from Mod Organizer, two folders need to be deleted from Mod Organizer's plugins folder, pluginfinder and data\pluginfinder. Once deleted, Plugin Finder will no longer be installed the next time Mod Organizer starts.

If Mod Organizer is installed to C:\Mod Organizer\ then these folders are located at C:\Mod Organizer\plugins\pluginfinder\ and C:\Mod Organizer\plugins\data\pluginfinder\
